Soooo... what's the difference between hashes and arrays?

In life, you will come across situations where you would want to store information that you want to access it later on. With a computer, you can store a massive amount information in place called "database" and access it with a "key". For the purpose of illustrating how storing and accessing information works in Ruby, let's assume we would want to store information about your (or my) favorite things!

In Ruby, there are two ways of storing information: array and hash. Think of array as an ordered list of the information you store, and hash as an unordered list. For array, you will access information by the order it is entered into the database (a number). As for hash, since it's an unordered list, you will access information with key.

Array: The Ordered List

favorite_food = ['sushi','pizza','cake'] Above is an example code of how you would store information in the array. Since the data is stored in an ordered list, if you want to retrieve information of, let' say, sushi, you would type... favorite_food [0] Notice that for array, the indexing starts with "0", so the first piece of data in an array starts with the index of ZER0. And if you want to retrieve "cake", you'll type... favorite_food [2] Easy right?

Hash: The UNordered LIst

Data stored using the hash method is not of any particular order. So to access the data, you need a specific key that points to the piece of information you want to retrieve. For example, to store information, you would do...

favorite_stuff = { "color" => "blue", "food" => "sushi", "number" => 0 } Now the data of some of my favorite stuff is stored in the hashes. In this case, the "key" to the answer of "blue" is color. So if someone else wants to know what's my favorite color, he/she will just type... favorite_stuff["color"] Not too bad huh?

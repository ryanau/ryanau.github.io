What's a Ruby Class?

When you first come into contact with Ruby class, you might be confused by what a Ruby Class is and what it actually does. To illustrate what a Ruby Class is, let's go with an example!

The Ticket Box Example

Imagine you're the software engineer running a box office at the Berkeley football stadium, and your job is to process and manipulate the data you have in your database, like modifying the ticket information you have. Sure, you can write a method for each ticket you have, but there is a better way of doing this, especially if you're a software engineer!

What Ruby offers you, is you can create a ticket Class, so every ticket (think of it as an object the database creates in the thousands) will have certain behavior (or method) you have already predefined the moment it is created. This way, you don't have to write a specific method, e.g. checking the price, modifying the date etc. for each ticket. You only have to write it to the ticket Class. To create a Ticket Class and write a simple method within the ticket Class:

class Ticket def type puts "Student Ticket" end end ticket = Ticket.new ticket.type

This will return

Student Ticket

You've just created a Ticket Class. Every time you create a new object using the .new method, you create a new object within the Ticket Class that has a certain behavior. The behvaior we've defined here is the .type method, so if we run the .type method, it will show us the type of the ticket: Student Ticket.

Instance Variables

Obviously, there's so much more you can do with Ruby Class. One common use is to store information when you create the object. In this case, when we create the ticket object, we would also want to store the price information, so we can access the price of the ticket later on.

class Ticket def initialize(input) @price = input end def get_price puts "Returning the price of the ticket:" puts @price end end ticket = Ticket.new(15) ticket.get_price

which will return

Returning the price of the ticket: 15

Now we've just created the Ticket class with 2 methods and 1 instance variable. The first method we created is the .initialize method with a specific argument of (15). When we do this, we established a specific "starting behavior" to the ticket object when we create it. In this case, 15 is the input (price) we give to every newly created Ticket object. So if we initialize (.new) on the ticket object with the argument of (15), we're basically saying that a new ticket has been issued with a price of $15. To store that information for later use, we stored it into a instance variable.

The instance variable is indicated with the @ sign, and we defined the instance variable to be @price with the input of (15) in this case. Later on, we defined another method called .get_price. Here, we're simply going to return a message and the instance variable of @price we established earlier. So in this case, the price of 15 has been returned becuase we've stored it earlier as an instance variable when we initialize the ticket object.

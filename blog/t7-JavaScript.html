A (Very) Brief Look at Ruby Hashes vs. JavaScript Objects

There are many fundamental differences between how Ruby and JavaScript do something. Today we'll take a look at Ruby Hashes and JavaScript Objects.

Ruby Hashes

In Ruby, hashes can be seens as an unorderlist that resembles a dictionary. There's a key and a value in every key-value pair that can store data. For example, if you want to store data about your family's favorite fruit, you can do:

Dad => Watermelon Mom => Orange Younger Sister => Coconut Older Brother => Mango

JavaScript Objects

A JavaScript object is a variable that you can store information in. It is similar to Ruby hashes in some ways but it offers you more flexibility, because it can store functions as well. Let's say you are defining an object that is a car. To define a JavaScript object, you do:

var car = { make: "toyota", year: 2012, color: "black" };

As mentioned, you can also store functions within the variable. For example, if we want to make a function that tells what color is the car, we can add this:

var car = { make: "toyota", year: 2012, color: "black" color_of_car: function () {return "This car is " + this.color + "color."} };

Now when we call the function by writing:

car.color_of_car

The sentence: "This car is black color." will print.

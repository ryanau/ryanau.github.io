A Brief Overview of 3 Main Sorting Methods: Bubble Sort, Quick Sort and Merge Sort

Have you ever wondered how do some programming languages sort data, e.g. in an array? Today, we'll take a look at 3 different sorting methods, which are bubble sort, quick sort and merge sort.

Bubble Sort

Bubble sort is a simple sorting algorithm that repeatedly steps through the list to be sorted. It compares each pair of adjacent items and simply swaps them if they are in the wrong order. Below is an example of how this sorting works:

As you can see, this sorting method can take a while since the algorithm has to go through each pair of elements. Also, it has to do this operation enough times in order to ensure all the pairs are sorted. In fact, it has to iterate through the list n times and compare n-pairs in order to complete the sorting. So on average, bubble sort is said to have O(n^2) complexity, meaning that it takes n^2 amount of time to complete the sorting. This is pretty horrible when compared to the other sorting methods.

Quick Sort

Quick sort is said to be a divide and conquer sorting algorithm, where you first choose a pivot value, usually the middle element in the array. Then, you rearrange all elements in such a way so that all elements that are less than the pivot go to the left part of the array. The larger elements will then go to the right of the pivot. Merge Sort

Merge sort first divides the list into sublists, down to where each sublist is a pair. It then compares the two elements and put them into place. Once that's done, it merges the different sublists into a bigger list.
